Aim: The aim of this lab session is to provide students with practical experience in
understanding and implementing Wi-Fi security management techniques on computer
networks.
Tools Required:
1. Wi-Fi-enabled laptops or computers
2. Wi-Fi access points
3. Network security software (e.g., Wireshark, Aircrack-ng)
4. Documentation and reference materials on Wi-Fi security protocols and standards

Theory:
Before conducting the experiment, students should review the following theoretical concepts:
1. Wi-Fi Security Protocols:
 WEP (Wired Equivalent Privacy) – Weak due to cryptographic flaws.
 WPA (Wi-Fi Protected Access) – Improved security but still vulnerable.
 WPA2 (Wi-Fi Protected Access 2) – Uses AES encryption and is widely
used.
 WPA3 – Latest standard with improved encryption and security mechanisms.
2. Encryption Methods:
 WEP (RC4-based, weak security)
 TKIP (Temporal Key Integrity Protocol, improved but outdated)
 AES (Advanced Encryption Standard, strong security)
3. Authentication Mechanisms:
 PSK (Pre-Shared Key, used in WPA2-PSK, WPA3-SAE)
 EAP (Extensible Authentication Protocol, used in enterprise networks)
 802.1X (Network access control framework used with RADIUS servers)
4. Common Wi-Fi Security Vulnerabilities and Threats:
 Man-in-the-middle attacks (MITM)
 Evil Twin attacks (Fake access points)
 Deauthentication attacks

 Brute-force attacks on passwords
 Packet sniffing
5. Best Practices for Wi-Fi Security:
 Use WPA3 or WPA2 with AES encryption.
 Implement strong passwords and multi-factor authentication.
 Disable WPS (Wi-Fi Protected Setup).
 Use MAC address filtering and hidden SSIDs.
 Regularly monitor and analyze network traffic for anomalies.
Experimental Procedure:
Step 1: Setup
1. Set up a Wi-Fi access point and connect Wi-Fi-enabled computers to it.
2. Ensure that the network is working correctly and all devices are configured properly.
3.
Step 2: Configuration
1. Access the router’s admin panel (e.g., 192.168.1.1 or 192.168.0.1).
2. Configure the following security settings:
 SSID (Wi-Fi name) – Change from default.
 Encryption Method – Set to WPA2-AES or WPA3 if supported.
 Authentication Mode – Use PSK (for home use) or 802.1X (for enterprise
networks).
 Wi-Fi Password – Set a strong password with at least 12 characters.
3. Disable WPS (Wi-Fi Protected Setup) to prevent brute-force attacks.
4. Enable MAC address filtering to allow only authorized devices.
Step 3: Monitoring and Analysis
1. Using Wireshark for Packet Capture:
 Open Wireshark on a laptop connected to the network.
 Start capturing packets on the Wi-Fi interface.
 Apply filters to analyze security threats (e.g., wlan.fc.type_subtype == 0x0c
for deauthentication attacks).
 Identify possible unauthorized connections or suspicious activities.
2. Using Aircrack-ng for Security Testing (Only in a Controlled Environment)
 Run airmon-ng start wlan0 to enable monitor mode.
 Use airodump-ng wlan0mon to scan nearby Wi-Fi networks.
 Attempt to test brute-force attacks (only for educational purposes and with
permission).

Step 4: Implementing Security Enhancements
1. Enable strong encryption (WPA2-AES or WPA3).
2. Implement MAC filtering to restrict access.
3. Set up firewall rules to prevent unauthorized access.

4. Test security measures by simulating attacks (e.g., deauthentication attack using
Aireplay-ng) and analyzing responses.
5. Verify if security configurations prevent unauthorized access.
Step 5: Documentation and Output Generation
At the end of the experiment, students should generate:
1. Wi-Fi Security Configuration Report – Screenshots of the access point settings and
security configurations.
2. Wi-Fi Traffic Analysis Report – Packet captures and security vulnerabilities
identified using Wireshark.
3. Security Implementation Report – Details of encryption, authentication, and
security measures applied.
