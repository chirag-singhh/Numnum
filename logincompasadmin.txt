Aim: Log into the Computer System as an Administrator and Check Security Policies

Objective:
The primary aim of this experiment is to understand the importance of security policies
within computer systems, learn how to log in as an administrator, identify and analyze
security policies, and interpret the implications of these policies for system security and user
access.

Materials Required:
1. Computer System with administrator access.
2. Operating System with security policy settings (e.g., Windows Group Policy Editor,
Linux Security Configuration).
3. Documentation tools for recording observations and findings (e.g., a notebook,
document editor, or security auditing software).

Theory:
Security policies are a set of guidelines and rules governing the security of a computer
system. They dictate various system behaviors like:
- User Access Levels: Who can access the system and at what level (e.g., regular user,
administrator).
- Password Requirements: What minimum password complexity should be followed.
- Network Configurations: Settings for secure communication over networks.
- File Permissions: Defining what data or files users can access or modify.
- Audit Policies: Tracking user activities to monitor security and detect unauthorized
activities.

By logging in as an administrator, users gain higher privileges to make changes to these
policies, view configurations, and implement security measures for the system.

Experiment Procedure:
Step 1: Preparation
1. Ensure System is Ready:
 Confirm that you have administrative access to the computer.
 Check if the operating system used has security policy settings (e.g., Windows
Group Policy Editor, Linux security configurations).
 Prepare a system to record your observations (such as a document editor or
note-taking tool).
2. Understand Security Policies:
 Familiarize yourself with common security policy settings, such as password
strength, access control, login audit policies, and network restrictions.
 Learn the specific tool you will be using (Group Policy Editor on Windows,
iptables or SELinux configurations on Linux).
Step 2: Log into the System as an Administrator
1. For Windows Operating System:
 Boot up the system and wait for the login screen.
 Enter the administrator credentials (username and password).
 Ensure you are logged into an account with administrative privileges. This
will allow you to make system-wide changes.

2. For Linux Operating System:
 Open the terminal.
 Use the sudo command to gain administrator privileges (for example: sudo su
or sudo -i).
 Enter the administrator password when prompted.

Step 3: Access Security Policies
1. For Windows:
 Open the Group Policy Editor by typing gpedit.msc in the Run dialog (Win +
R).

 Navigate through the settings to access policies related to Account Policies,
User Rights Assignment, Local Policies, and Security Options.

2. For Linux:
 Check security settings in /etc/security/ (for PAM configurations) and use the
iptables command to review firewall settings.
 Look into SELinux or AppArmor configurations for more advanced security
policies.
3. Explore Policies:
 Review settings such as password length requirements, account lockout
policies, audit policies, and network restrictions.
 Examine settings related to user access control, administrator privileges,
and permission settings.
 Pay attention to any encryption or user authentication policies.

Step 4: Analyze Security Policies
1. Password Policies:
 Check if password complexity is enforced (e.g., minimum length, special
characters, etc.).
 Investigate the account lockout policies (e.g., after how many failed login
attempts will the account be locked?).

2. User Access Control:
 Identify who has administrator privileges and who is a standard user.
 Examine user rights assignments, such as who can access critical system
files or change security settings.

3. Network Policies:
 Look for any firewall settings or IP filtering policies.
 Review any network-related restrictions, such as preventing certain
applications from accessing the network.

4. Audit Policies:
 Ensure that logging and audit trails are enabled for important actions (e.g.,
login attempts, file access).
 Determine the level of monitoring configured to track administrator
activities.

Step 5: Record Findings and Document Observations

 Take notes on all the policies and settings you find, including:
 Specific security settings (password requirements, user rights, etc.).
 Any discrepancies you notice, such as missing password complexity
requirements or open ports in the firewall.
 Vulnerabilities or potential areas of improvement.
 Draw conclusions about the strength of the system&#39;s security based on the policies
implemented.
Step 6: Evaluate the Effectiveness of Security Policies
 Analyze the effectiveness of each policy you observed:
 Are the password policies robust enough to prevent brute force attacks?
 Is the system logging enough information for auditing user activities?
 Are network configurations secure against potential intrusions?
 Are any security gaps present (e.g., weak user authentication or missing
encryption)?

Step 7: Suggest Improvements (If Any)
 Based on your analysis, recommend improvements to existing security policies.
 For example, you may suggest stronger password policies, additional user
activity logging, or the implementation of a stricter firewall.
