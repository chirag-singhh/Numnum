Aim: Setting, Configuring, and Managing Three Password Policies in the Computer (BIOS,
Administrator, and Standard User).
Tools:
1. Computer with BIOS access
2. Operating system with administrative privileges
3. User account management tools (e.g., Group Policy Editor for Windows, Linux user
management tools)
Theory: Password policies play a crucial role in maintaining the security of computer
systems by enforcing strong authentication measures. A password policy typically includes
rules and requirements for creating, managing, and using passwords effectively. It helps
prevent unauthorized access, data breaches, and other security incidents. In this experiment,
we will focus on configuring password policies at three different levels: BIOS, Administrator,
and Standard User.
1. BIOS Password Policy:
 The BIOS password is set to restrict access to the computer’s firmware
settings.
 Prevents unauthorized users from changing system configurations.
 Can be set up by accessing BIOS/UEFI during system boot (commonly by
pressing F2, F10, or Del key).
2. Administrator Password Policy:
 Ensures that only authorized users have administrative privileges on the
system.
 Can be configured through the Windows Local Security Policy or Linux user
management tools.
 Enforces password complexity, expiration, and account lockout policies.
3. Standard User Password Policy:

 Applies to regular users who do not have administrative privileges.
 Restricts access to sensitive settings and system modifications.
 Can be managed using User Account Settings in Windows or Linux.

Procedure:
1. Configuring BIOS Password:
i. Restart the computer and enter the BIOS/UEFI setup.
ii. Navigate to the Security settings.
iii. Set and confirm a BIOS password.
iv. Save changes and exit BIOS.
2. Configuring Administrator Password Policy:
secpol.msc
i. Open the Local Security Policy in Windows (or use passwd in Linux).
ii. Navigate to Account Policies &gt; Password Policy.
iii. Set minimum password length, complexity requirements, and expiration
period.
iv. Apply changes and restart the system.
3. Configuring Standard User Password Policy:
i. Create a standard user account if not already available.
ii. Set up a strong password policy using the Group Policy Editor (Windows) or
chage command in Linux.
iii. Restrict password reuse and set expiration rules.

Output:
1. Configured password policies for BIOS, Administrator, and Standard User accounts.
2. Documentation detailing the configured policies and their respective requirements.
3. Observations on the impact of different password policies on system security and
usability.
