# **Experiment: Firewall Configuration on Linux and Windows**

## **Aim**

To configure and analyze firewall rules on both **Linux** and **Windows** operating systems to enhance network security.

---

## **Tools Required**

1. Computer or Virtual Machine running **Linux** (Ubuntu/Kali).
2. Computer or Virtual Machine running **Windows 10/11**.
3. **UFW (Uncomplicated Firewall)** for Linux.
4. **Windows Defender Firewall with Advanced Security** for Windows.

---

## **Theory**

A **firewall** is a network security system that monitors and controls incoming and outgoing network traffic based on predefined security rules. It helps prevent unauthorized access, reduces cyber threats, and enforces network security policies.

Firewalls are essential for protecting systems from hackers, malware, and unauthorized network connections.

---

## **Types of Firewalls**

1. **Packet Filtering Firewall**
   Analyzes individual data packets based on IP address, port number, and protocol.

2. **Stateful Inspection Firewall**
   Tracks the state of active connections and allows only legitimate traffic.

3. **Application Layer Firewall**
   Inspects traffic at the application level to provide advanced security.

---

## **Firewall Tools Used**

* **Linux:** Uses **UFW (Uncomplicated Firewall)** to easily manage firewall rules.
* **Windows:** Uses **Windows Defender Firewall** to configure inbound and outbound traffic rules.

---

# **Procedure**

## **Step 1: Configure Firewall on Linux using UFW**

### **1. Enable UFW**

```bash
sudo ufw enable
```

**Explanation:**
This command enables the firewall. Until the firewall is enabled, it does not provide any protection. Once enabled, the system starts monitoring network traffic.

---

### **2. Allow SSH (Port 22)**

```bash
sudo ufw allow 22/tcp
```

**Explanation:**
This command allows SSH connections on port 22. SSH is used for remote login into the system.

---

### **3. Allow HTTP (Port 80) and HTTPS (Port 443)**

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

**Explanation:**

* Port **80** allows HTTP traffic (websites).
* Port **443** allows HTTPS traffic (secure websites).
  These ports are necessary when hosting or accessing web services.

---

### **4. Block All Incoming Traffic Except Allowed Rules**

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
```

**Explanation:**

* `deny incoming` blocks all incoming traffic except the ports that are explicitly allowed.
* `allow outgoing` allows the system to send outgoing traffic such as browsing and updates.

This improves system security by restricting unauthorized access.

---

### **5. Check Firewall Status**

```bash
sudo ufw status verbose
```

**Explanation:**
This command displays the firewall status and shows which ports are allowed or blocked.

---

# **Step 2: Configure Firewall on Windows**

### **1. Open Windows Defender Firewall with Advanced Security**

* Press **Windows Key**
* Search **Windows Defender Firewall with Advanced Security**
* Open the application.

---

### **2. Navigate to Rules**

* Click **Inbound Rules**
* Click **Outbound Rules**

These sections allow configuration of incoming and outgoing traffic.

---

### **3. Create New Rules to Allow Ports**

Create rules to allow traffic on:

* **Port 22 (SSH)**
* **Port 80 (HTTP)**
* **Port 443 (HTTPS)**

**Steps:**

1. Click **New Rule**
2. Select **Port**
3. Choose **TCP**
4. Enter port number (22, 80, 443)
5. Select **Allow the connection**
6. Apply rule to all profiles
7. Give rule a name
8. Click **Finish**

---

### **4. Block All Other Traffic**

Set the default policy to **block all incoming traffic** except allowed rules.
This ensures only permitted services can communicate.

---

### **5. Verify Firewall Rules**

* Open firewall settings
* Check **Inbound and Outbound Rules**
* View **Firewall Logs** to confirm rules are applied correctly.

---

# **Result**

Firewall rules were successfully configured on both Linux and Windows systems. Necessary ports such as **SSH (22), HTTP (80), and HTTPS (443)** were allowed, while unauthorized incoming traffic was blocked, improving overall system security.

---

# **Conclusion**

This experiment demonstrated how firewall rules can be configured using **UFW in Linux** and **Windows Defender Firewall** in Windows. Proper firewall configuration helps protect systems from unauthorized access and improves network security.
